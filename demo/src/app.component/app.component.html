KX-MODALS
  • Angular modals done right.

Configuration

Configuration for the kx-modals library to work is really simple, you just include the KxModalModule into your root module (usually app.module.ts) and add it to the imports list, as such:

app.module.ts inclusion example 
import {{ '{ '}} KxModalModule {{ '}' }} from 'kx-modals';

@NgModule({{ '{' }})
	imports: [
		KxModalModule.forRoot({{ '{' }}
			// configuration
		{{ '}' }}),
		...
	],
	...
{{ '}' }})
export class AppModule {{ '{ }' }}

After having done this, make sure you've included the <kx-modal-container> somewhere in your application, though preferably somewhere in your root component. After that is done, you can use the KxModalService anywhere in your application to create a modal component. The only thing your component has to comply to, is that it is derived from the KxModalBaseComponent class.

For an example of how to use this, just look at one of the modal examples below.

Confirmation modals

Modals that would normally require you to confirm an action, such as deleting an item.
